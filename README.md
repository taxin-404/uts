# 🕌 উদ্দীপ্ত তরুণ সংঘ — Islamic Dawah & Research Article platform

একটি পূর্ণাঙ্গ, সুরক্ষিত এবং প্রতিক্রিয়াশীল দাওয়াহ ও মানবিক সাহায্য কার্যক্রম প্ল্যাটফর্ম। এই অ্যাপ্লিকেশনটি Reddit এবং chintaporadh.com-এর অনুপ্রেরণায় তৈরি করা হয়েছে, যেখানে সংগঠনের সমাজসেবামূলক উদ্যোগে স্বচ্ছতা ফুটিয়ে তুলতে রিয়েল-টাইম ভোটদান এবং হায়ারার্কিকাল (একটি কমেন্টের ভেতর আরেকটি ডেসিমাল রিপ্লাই) কমেন্টিং সিস্টেম যুক্ত করা হয়েছে।

উন্নয়ন ও হোস্টিং খরচের বিষয়টি মাথায় রেখে, পুরো আর্কিটেকচারটি এমনভাবে ডিজাইন করা হয়েছে যাতে আপনি সম্পূর্ণ **১০০% বিনামূল্যে (Free Tier-এ)** একটি লাইভ প্রোডাকশন সাইট গড়ে তুলতে পারেন।

---

## 🛠️ প্রযুক্তিগত বৈশিষ্ট্যসমূহ (Core Architecture)
1. **Frontend:** React 19 + Tailwind CSS v4 + Lucide Icons.
2. **Backend:** Node.js Express Server serving static bundles in Production.
3. **Database:** PostgreSQL (Supabase Free Tier Cloud Database Engine).
4. **Media Storage:** ImgBB Free Client API Gateway for unlimited content upload.
5. **Authorization / RLS:** Row-level Security checks directly verifying JWT tokens inside Supabase context.

## 🏗️ অবজেক্ট ডাটাবেস ও মাইগ্রেশন গাইড (`schema.sql`)
অ্যাপ্লিকেশনটির সকল টেবিল এবং রো-লেভেল অ্যাক্সেস কন্ট্রোল পলিসি (RLS) কার্যকর করার জন্য প্রদেয় `schema.sql` ফাইলটি তৈরি করা হয়েছে।

### টেবিলগুলোর সম্পর্ক ও কার্যাবলী
* **`profiles`**: ব্যবহারকারীদের তথ্য ও দায়িত্বের ভূমিকা (`user`, `admin`) এতে সংরক্ষিত থাকে। সুপাবেস সাইন-আপ হুকের সাথে ট্রিগার মেকানিজম যুক্ত থাকায় কোন গ্রাহক সাইন আপ করার সাথে সাথে স্বয়ংক্রিয়ভাবে তার পাবলিক প্রোফাইল তৈরি হয়।
* **`posts`**: প্রতিটি প্রবন্ধের শিরোনাম, মার্কডাউন লেখা, ImgBB ইমেজ লিঙ্ক এবং ভোটের সংখ্যা ট্র্যাক করে। অফিসিয়াল মডিউলগুলো (ফিড, কার্যক্রম, নির্দেশিকা) সাধারণ ইউজারদের জন্য রিড-অনলি করা হয়েছে।
* **`votes`**: ব্যবহারকারী এবং পোস্টের যৌথ চাবির মাধ্যমে ডুপ্লিকেট ভোট রোধ করে। একজন ব্যবহারকারী একটি পোস্টে শুধুমাত্র একটি আপভোট বা ডাউনভোটই প্রদান করতে পারবেন।
* **`comments`**: হায়ারার্কিকাল সেলফ-রেফারেন্সিয়াল ফরেইন কি (`parent_id`) ব্যবহারের মাধ্যমে অসীম গভীরতার কমেন্ট ও রিপ্লাই কাঠামো প্রিসেট করা আছে।

## 🚀 লোকাল সেটআপ সিকোয়েন্স (Local Setup Guide)

### ১. রিপোজিটরি ক্লোন ও প্যাকেজ ইনস্টলেশন
অনুকূল টার্মিনাল রানার ব্যবহার করে প্যাকেজগুলো ইনস্টল করুন:
```bash
# ডিপেন্ডেন্সি সংস্থাপন করুন
npm install
```

### ২. এনভায়রনমেন্ট ফাইল কনফিগারেশন
রুটে অবস্থিত `.env.example` ফাইলের নাম পরিবর্তন করে `.env` রাখুন এবং আপনার সংগৃহীত অ্যাক্সেস কীগুলো বসিয়ে দিন:
```env
# Supabase
SUPABASE_URL="https://your-project-id.supabase.co"
SUPABASE_ANON_KEY="eyJhbGciOi..."
SUPABASE_SERVICE_ROLE_KEY="eyJhbGciOi..."

# ImgBB (ছবি আপলোডের ফ্রি সার্ভিস)
VITE_IMGBB_API_KEY="your-imgbb-free-api-key"
```

### ৩. লোকাল ডেভেলপমেন্ট সার্ভার বুট করা
```bash
# লোকাল ডেভেলপমেন্ট সার্ভার চালু করুন (Port 3000)
npm run dev
```
সার্ভারটি বুট হয়ে স্থানীয় কো-অর্ডিনেট সংকেত `http://localhost:3000` এ রানিং অবস্থায় চালু হবে।

## 🗃️ ডাটাবেস পাইপলাইন কনফিগারেশন (Supabase)

১. [Supabase Console](https://database.supabase.com) এ লগইন করে একটি নতুন ফ্রি প্রজেক্ট তৈরি করুন।
২. আপনার ড্যাশবোর্ড থেকে বামদিকের মেনুর **SQL Editor** অপশনে প্রবেশ করুন।
৩. প্রজেক্টের রুটে সরবরাহ করা `schema.sql` ফাইলের সম্পূর্ণ টেক্সট কপি করে SQL উইন্ডোতে পেস্ট করে দিন।
৪. **Run** বাটনে চাপ দিয়ে সফলভাবে এক্সিকিউট করুন। এটি আপনার ডাটাবেস টেবিল, আরএলএস পলিসি ও সাইন-আপ ট্রিগার ফাংশনগুলো ইনস্ট্যান্ট সচল করে দেবে।

### 👑 যেকোনো ব্যবহারকারীকে এডমিনে উন্নীত করার শর্টকাট ট্রিক (Admin Elevation Query)
সুপাবেস কুয়েরি ম্যানেজারে নিচের SQL কোডটি রান করে নির্দিষ্ট ব্যবহারকারী প্রোফাইলকে সাধারণ ইউজার থেকে **Supervisor/Admin** পদে উন্নীত করতে পারেন:
```sql
-- আপনার কাঙ্ক্ষিত ইউজারের আইডি ব্যবহার করে প্রোফাইল আপডেট করুন
UPDATE public.profiles 
SET role = 'admin'::user_role 
WHERE email = 'admin@yourdomain.com';
```

## 🌐 জিরো-কস্ট প্রোডাকশন হোস্টিং টপোলজি (100% Free Hosting)

### ১. ডাটাবেস হোস্টিং (Supabase Free Tier)
* **পরিষেবা:** Supabase PostgreSQL Database Engine.
* **সীমাবদ্ধতা:** ৫০০ MB স্পেস এবং ২ GB ইজি ব্যান্ডউইথ (ব্যক্তিগত বা ক্ষুদ্র প্রচারণার জন্য অত্যন্ত চমৎকার)।
* **কনফিগারেশন:** সুপাবেস ড্যাশবোর্ডের `Settings > API` মেমোরি ট্যাব থেকে *Project URL*, *Anon key* এবং *Service Role key* সংগ্রহ করবেন।

### ২. ব্যাকএন্ড এপিআই ও কম্পিউটেশন ক্লাউড (Render.com Free Services)
* **পরিষেবা:** Render Free Web Servies (Web Service).
* **কনফিগারেশন ধাপসমূহ:**
  1. আপনার গিথাব (GitHub) অ্যাকাউন্টের সাথে Render সংযুক্ত করুন এবং এই রিপোজিটরি চুজ করুন।
  2. Build Command হোন: `npm run build`.
  3. Start Command দিন: `npm run start` (বা `node dist/server.cjs` ব্যবহার)।
  4. পরিবেশ বিন্যাস বা **Environment Variables** সেটিংসে গিথাবের সিক্রেট হিসেবে `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` যোগ নিশ্চিত করুন।
  5. *বিঃদ্র:* রেন্ডার ফ্রি প্ল্যানে অব্যবহৃত থাকলে প্রথম রিকোয়েস্টে ব্যাকএন্ড বুট হতে প্রায় ৫০ সেকেন্ড সময় নিতে পারে, এটি ফ্রি ক্লাউডের স্বাভাবিক আচরণ।

### ৩. ফ্রন্টএন্ড কোড বেস বুস্ট (Vercel / Netlify)
যেহেতু অ্যাপটি এক্সপ্রেসের মিডলওয়্যার মোডে সম্পূর্ণ স্বয়ংসম্পূর্ণ ভাবে চলে, আপনি এটি রেন্ডারে এক্সপ্রেস সার্ভারের সাথে একসাথেই রান করতে পারেন। তবে আপনি যদি সম্পূর্ণ ক্লায়েন্ট সাইড এপিআই প্রক্সি তৈরি করতে চান:
* **হোস্টিং মেকানিজম:** Vercel বা Netlify ফ্রি ড্যাশবোর্ডে কানেক্ট করে `dist` ডিরেক্টরিটিকে রেন্ডার পাথ হিসেবে দিয়ে খুব সহজেই লাইভ ইউআরএল ক্লাউড জেনারেট করতে পারবেন।

### ৪. ফ্রী ইমেজ ক্লাউড সেটআপ (ImgBB Setup)
* **উদ্দেশ্য:** মিডিয়া বা বড় ছবি সরাসরি ডাটাবেসে রাখলে সার্ভারের স্টোরেজ দ্রুত শেষ হয়ে যাবে। কিন্তু ImgBB ব্যবহার করলে ডাটাবেস মাত্র কয়েক বাইট লিঙ্ক ডাটা সেভ করবে।
* **ধাপসমূহ:**
  1. [ImgBB](https://imgbb.com/) সাইটে যান এবং একটি ফ্রী অ্যাকাউন্ট রেজিস্ট্রেশন করুন।
  2. তাদের ডেভেলপার এপিআই পেজ (`https://api.imgbb.com/`) থেকে একটি ফ্রী **API Key** সংগ্রহ করুন।
  3. সংগৃহীত কীটি আপনার সেটিংস প্যানেলে `VITE_IMGBB_API_KEY` হিসেবে যুক্ত করে দিন।
  4. এরপর এডমিন পেজ থেকে ছবি নির্বাচন করলে তা সরাসরি সেকেন্ডের মধ্যে তাদের CDN-এ আপলোড হবে এবং আপনার দাহওয়াহ প্রবন্ধের সাথে যুক্ত হয়ে যাবে।

## 🔑 গুগল ও-অথ লগইন কনফিগারেশন (Google OAuth Setup Guide)

প্ল্যাটফর্মে নিরাপদ গুগল সিঙ্গেল সাইন-অন (Google Login) সচল করতে নিচের ধাপগুলো ক্লিয়ারলি সম্পাদন করুন:

### ধাপ ১: Google Cloud Console থেকে ক্রেডেনশিয়াল তৈরি
1. **Google Cloud Console (`https://console.cloud.google.com`)** এ প্রবেশ করে আপনার জিমেইল অ্যাকাউন্ট দিয়ে সাইন-ইন করুন।
2. একটি নতুন প্রজেক্ট তৈরি করুন (বা পূর্ববর্তী প্রজেক্ট সিলেক্ট করুন)।
3. সার্চ বারে **API & Services** লিখে সার্চ করুন অথবা মেনু থেকে **APIs & Services > Credentials** এ যান।
4. ক্রেডেনশিয়াল তৈরি করার আগে **OAuth consent screen** লিঙ্কে ক্লিক করে প্রয়োজনীয় তথ্য (যেমন: App name, User support email) দিয়ে প্রাথমিক কনফিগারেশন সেট করুন এবং Scope হিসেবে `email` ও `profile` অ্যাডিশন করুন।
5. এরপর **+ CREATE CREDENTIALS** বোতামে ক্লিক করে **OAuth client ID** সিলেক্ট করুন।
6. **Application type** হিসেবে নির্বাচন করুন **Web application**।
7. **Authorized JavaScript origins** বক্সে আপনার লাইভ হোস্টেড অ্যাপ বা ডেভ সার্ভার অ্যাপ ইউআরএল এবং রিডাইরেক্ট ইউআরএল যোগ করুন:
   - `http://localhost:3000` (লোকাল ডেভেলপমেন্টের জন্য)
   - `https://your-project-id.supabase.co` (সুপাবেসের জন্য)
8. **Authorized redirect URIs** বক্সে সুপাবেস ড্যাশবোর্ড থেকে প্রাপ্ত স্পেসিফিক ও-অথ কলব্যাক লিঙ্কটি প্রবেশ করান:
   - এটি সাধারণত এই ফরম্যাটে থাকে: `https://<YOUR-SUPABASE-PROJECT-REF>.supabase.co/auth/v1/callback`
9. **Create** বাটনে প্রেস করলে গুগল আপনাকে একটি **Client ID** এবং **Client Secret** পপ-আপ বক্সে প্রদর্শন করবে। এই দুটি কী তাৎক্ষণিকভাবে কপি করে রাখুন।

### ধাপ ২: Supabase ড্যাশবোর্ডে ও-অথ সংযোগ চালু করা
1. আপনার **Supabase Dashboard (`https://supabase.com/dashboard`)** প্রজেক্টে যান।
2. বামদিকের মেনু বার থেকে **Authentication > Providers** এ ক্লিক করুন।
3. প্রোভাইডারদের লম্বা লিস্ট থেকে **Google** খুঁজে বের করে তা এক্সপ্যান্ড (প্রসারিত) করুন।
4. **Enabled Google Provider** টগলটি সুইচ অন (সচল) করে দিন।
5. গুগল ক্লাউড কনসোল থেকে ধাপে কপি করা **Client ID** এবং **Client Secret** যথাক্রমে নির্ধারিত বক্সে পেস্ট করুন।
6. নিচে স্ক্রল করে **Save** বোতামে ক্লিক করুন।

ব্যাস! আপনার প্ল্যাটফর্মটি এখন সম্পূর্ণ ক্লাউড অথেনটিকেশনের মাধ্যমে জেনুইন ইন্টিগ্রেশনে সচল। কোন ব্যবহারকারী মডালে অবস্থিত "Google-লগইন" বোতামে ক্লিক করলে সরাসরি তার গুগলের ভ্যারিফাইড নাম ও অ্যাভাটার ডেটা ডাটাবেসের `public.profiles` টেবিলে স্বয়ংক্রিয় সিঙ্ক হয়ে চলে যাবে।
